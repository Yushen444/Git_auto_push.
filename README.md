# Git自动推送程序

这个项目包含一个自动化的Git推送程序，可以自动执行Git操作并记录推送历史。

## 功能特性

- 自动以管理员身份运行Git命令
- 添加当前文件夹所有文件到本地仓库分支
- 使用当天日期作为提交信息（例如："2026-02-03"）
- 推送到GitHub远程仓库
- 生成Push-history.md文件记录推送历史（时间精确到秒，包含IP地址和提交信息）
- 自动拉取远程更改，避免冲突
- 程序可移植，使用相对路径

## 目录结构

```
├── git_auto_push/          # 程序文件夹
│   ├── git_auto_push.exe   # 主程序exe文件
│   ├── git_auto_push.py    # Python源码
│   ├── build/             # 打包中间文件
│   ├── dist/              # 打包输出目录
│   └── run_as_admin.bat   # 管理员权限运行脚本
├── run_git_push.bat       # 快捷方式批处理文件
├── Push-history.md        # 推送历史记录
└── .gitignore            # Git忽略文件配置
```

## 使用方法

1. 直接双击运行 `run_git_push.bat` 文件
2. 程序会自动查找Git仓库根目录
3. 先拉取远程更改，再将本地更改提交并推送
4. 在 `Push-history.md` 文件中记录推送历史

## 程序原理

- 程序会自动查找包含`.git`文件夹的根目录
- 检测所有未提交的更改并添加到暂存区
- 使用当前日期作为提交信息
- 执行提交和推送操作
- 记录推送时间、IP地址、提交信息等到历史文件

## 注意事项

- 确保已经正确配置了Git和GitHub远程仓库
- 需要有推送远程仓库的权限
- 程序文件已被添加到`.gitignore`，不会推送到远程仓库
- 推送历史文件会包含时间、IP和提交信息
